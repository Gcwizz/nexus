## Vision

### 10x Check
The PRD describes a linear pipeline — connect, map, optimise, spec, build, migrate. Runs once. The 10x version is a Living Business Operating System: connectors stay live, the ontology continuously updates, process maps drift-detect against reality, the AI optimiser is a persistent copilot, and the software evolves with the business. The platform doesn't just build software — it becomes the company's brain. Every new SME makes the platform smarter through cross-org learning (deferred). The economics improve with every customer — the opposite of consulting.

### Platonic Ideal
The SME CEO connects tools on Monday. By Tuesday they're walking through a 3D model of their entire business — spinning it like a globe, zooming into departments like Google Earth into streets. Red zones for broken things, green for flowing. By Wednesday they're dragging process blocks around with their management team, watching AI show ripple effects in real-time. Thursday they approve. Friday they have working software with all their data. It doesn't feel like a tech project. It feels like the business upgraded itself.

## Architecture Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Repo structure | Turborepo monorepo | 9 modules sharing data contracts need atomic updates |
| Graph database | Neo4j (Cypher) | Industry standard, best DX, Claude knows it well |
| Ontology format | Neo4j native + typed JSON | Simpler than JSON-LD, less token waste, export to RDF when needed |
| Graph coupling | Ownership Model | Module 2 owns ontology writes, Module 4 owns process writes, Module 6 owns target-state writes. Others read via versioned APIs |
| LLM security | P0 shared infrastructure | Sanitisation + structured output + per-org isolation. Non-negotiable |

## Security Requirements (P0)
- LLM prompt injection defence: data sanitisation layer + structured output enforcement + per-org context isolation
- IDOR protection: middleware-level org_id validation on every endpoint
- File parsing sandboxing: Apache Tika in sandboxed container with resource limits
- OWASP scanning: mandatory for all Module 8 generated code before deployment
- Audit logging: every data access, LLM call, and ontology mutation

## Critical Error Paths Resolved
1. SchemaEvolutionError → Schema version detection + validation layer
2. CyclicSyncError → Cycle detection with causation_chain circuit breaker
3. LLMRefusalError → Detect refusal, retry with anonymised data
4. ContextOverflowError → Chunk by source, merge pass
5. AgentDeadlockError → DAG dependency resolution + timeout watchdog
6. TokenBudgetError → Budget tracking at 80% threshold + checkpoint
7. RollbackFailureError → Transaction-based migration batches
